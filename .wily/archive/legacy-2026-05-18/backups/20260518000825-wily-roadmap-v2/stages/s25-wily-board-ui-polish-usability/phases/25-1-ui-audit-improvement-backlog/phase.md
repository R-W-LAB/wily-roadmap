# Phase 25-1: UI audit and improvement backlog

## Purpose

Inspect the current Wily Board UI as it is actually used and turn the friction into a bounded implementation backlog.

## Acceptance

- Browser screenshots capture Hub and repo detail at desktop and mobile widths.
- The audit identifies concrete scanability, navigation, live-feedback, responsive, and accessibility issues.
- Each proposed improvement has acceptance criteria and a verification method.
- The backlog is explicitly scoped so later phases do not drift into auth, schema, mutation, or new integration work.
